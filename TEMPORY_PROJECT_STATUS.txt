3) HOW IT WORKS (DEV FLOW)
============================================================

Workspace model:
- pnpm workspace with 3 projects:
  - apps/api (NestJS backend)
  - apps/web (React + Vite frontend)
  - packages/types (shared local package)

Dependency linking:
- Both apps depend on @task-flow/types as workspace:*.
- pnpm links local package automatically.

Root scripts:
- pnpm dev:api
  Starts Nest backend in watch mode.
- pnpm dev:web
  Starts Vite frontend dev server.
- pnpm dev
  Starts both in parallel with live streaming logs.
- pnpm build
  Builds all workspace packages/apps recursively.
- pnpm lint
  Runs lint recursively across workspace.

Backend runtime:
- Entry: apps/api/src/main.ts
- Default port: 3000 unless PORT env var set.

Frontend runtime:
- Entry: apps/web via Vite.
- Typical dev port: 5173; can auto-shift (e.g., 5174) if busy.

9) TROUBLESHOOTING MATRIX
============================================================

Issue: EADDRINUSE on backend startup
- Symptom:
  - Error listen EADDRINUSE :::3000
- Cause:
  - Another process already uses port 3000.
- Fix:
  - Stop process using 3000, or set PORT to another value.

Issue: Cannot resolve @/lib/utils in web build
- Symptom:
  - TS2307 for @/* imports
- Cause:
  - Alias paths placed in wrong tsconfig scope.
- Fix already applied:
  - paths moved to apps/web/tsconfig.app.json

Issue: TypeScript baseUrl deprecation warnings
- Symptom:
  - baseUrl deprecation diagnostics appear on unrelated options.
- Cause:
  - baseUrl present in tsconfig.
- Fix already applied:
  - baseUrl removed from workspace tsconfig files.

Issue: react-refresh only-export-components lint errors with shadcn
- Symptom:
  - Errors for exported variant constants (buttonVariants, badgeVariants).
- Cause:
  - Rule is strict for Fast Refresh patterns.
- Fix already applied:
  - allowExportNames configured in apps/web/eslint.config.js
